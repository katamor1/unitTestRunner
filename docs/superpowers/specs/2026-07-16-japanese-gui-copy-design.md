# 日本人ユーザー向けGUI文言見直し設計

## 目的

VS Code拡張の画面、コマンドパレット、設定画面、ダイアログ、通知で使う表示文言を、日本人ユーザーが操作結果と次の行動を判断しやすい表現へ統一する。

## 対象

- ワークフローパネルの簡易表示・詳細表示・設定セクション
- スイートのサイドバーパネルと一覧画面
- `package.json` のコマンド名、ビュー名、設定説明
- VS Codeの入力欄、ファイル選択、確認ダイアログ、通知、エラー
- 表示文言を固定値として検証しているTypeScriptテスト

## 対象外

- コマンドID、設定キー、JSONキー、ファイル名、レポート名、CLI引数
- `aria-label`、`role`、読み上げ専用通知など、スクリーンリーダー専用文言
- 処理順序、状態遷移、確認要否、エラー処理そのもの
- CLIと生成レポート本文の全面的な日本語化

## 文言方針

1. 操作ボタンは対象と動作を明示する。例: `選択を実行` は `選択したテストを実行` とする。
2. 完了・実行中・失敗の通知は、何が起きたかを先に示す。
3. `workspace`、`manifest`、`status`、`dry-run` など、日本語UIで理解しにくい英単語は原則として日本語へ置き換える。
4. 製品固有の正式名称または実ファイル名である `Quick Check`、`Full Gate`、`function_dossier.md`、`test_case_design.json` などは必要な箇所で保持し、説明を日本語にする。
5. `GREEN` / `Not GREEN` は判定ロジックの概念として保持できるが、画面では `合格` / `不合格` を主表示とし、必要な箇所だけ括弧内に元用語を残す。
6. 確認ダイアログのボタンは `続行` ではなく、`ビルドを実行`、`テストを実行`、`全件テストを実行` のように結果が分かるラベルにする。
7. エラーは、可能な範囲で「できなかったこと」「原因または確認箇所」を含める。
8. 文中の英数字と日本語の不要な空白、表記揺れを解消する。`VS Code`、`Visual C++ 6.0`、`CLI` は表記を統一する。

## 画面別の主な変更

### 設定

- `プロジェクトルート` を `ソースのルートフォルダー` にする。
- `.dsw`、出力先、スイート定義、ビルド構成、VC6環境設定、CLI実行ファイルの役割を自然な日本語で説明する。
- `設定値:` を `保存されている設定:` にし、現在使用中の値との差が分かるようにする。
- 選択ボタンは `.dswファイルを選択`、`実行ファイルを選択` など対象を明示する。

### ワークフロー

- `出力workspace` を `出力ワークスペース` に統一する。
- `Quick Summary`、`Full Gate` などの固有名称には、日本語の目的説明を添える。
- `dry-run` は画面上で `事前確認` と表現し、必要に応じて括弧で `dry-run` を併記する。
- `保存待ち`、`直近エラー`、実行中表示を、状況と次の行動が分かる文へ変更する。

### テストスイート

- `manifest` を `スイート定義ファイル`、`status` を `実行結果`、`workspace` を `ワークスペース` にする。
- `広い一覧を開く` を `スイート一覧を開く` にする。
- 集計は `合格`、`不合格`、`実行済み`、`失敗` を使う。
- 空状態に、登録方法または次の操作を示す。

### コマンド・ダイアログ・通知

- コマンドパレットの英語タイトルを日本語化する。
- 入力欄のプロンプトとファイル選択タイトルを、入力対象が分かる表現にする。
- 実行確認は対象別の具体的なタイトル、本文、実行ボタンを使う。
- キャンセルをエラー扱いせず、必要な箇所では静かに終了する既存挙動を保持する。既存実装が例外を使う箇所は、この変更では制御フローを変えず文言のみ見直す。

## テスト

- ワークフローHTMLに新しい主要文言が含まれ、旧混在表記が含まれないことを検証する。
- 設定ビューモデルのラベル、説明、ボタン文言を検証する。
- スイートHTMLを純粋関数としてテストできる範囲で、見出し・集計・操作文言を検証する。
- `package.json` のコマンドタイトルと設定説明は既存のアダプター契約テストで検証する。
- TypeScriptコンパイル、拡張テスト、リポジトリ全体のCIを実行する。

## 完了条件

- 主要GUIに、説明なしの `workspace`、`manifest`、`status`、`Not GREEN`、英語コマンド名が残っていない。
- コマンドID、設定キー、データ形式、処理挙動に差分がない。
- スクリーンリーダー専用文言には変更を加えていない。
- 追加・更新したテストと既存CIが成功する。
