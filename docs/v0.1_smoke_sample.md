# v0.1 Smoke Sample

This sample exercises the fixture project in `tests/fixtures/vc6_project` and writes generated
artifacts outside the fixture source tree.

## Analysis And Test Design CLI Flow

```powershell
$env:PYTHONPATH = "$PWD\src"
$fixture = "$PWD\tests\fixtures\vc6_project"
$out = "$env:TEMP\unitTestRunner-smoke\Control_Update"

py -m unit_test_runner analyze-function `
  --workspace $fixture `
  --dsw "$fixture\Product.dsw" `
  --source src\control.c `
  --function Control_Update `
  --configuration "Win32 Debug" `
  --project Control `
  --out $out

py -m unit_test_runner build-probe --dossier "$out\reports\function_dossier.json" --dry-run
```

This flow is useful when checking the standard `build-probe --dossier` and
`generate-test-design --dossier` surfaces.

```powershell
py -m unit_test_runner generate-test-design --dossier "$out\reports\function_dossier.json"
```

## Finalized Dossier Review Flow

Use `--finalize-dossier` when the output is intended for human review or the VS Code adapter path.

```powershell
py -m unit_test_runner --json analyze-function `
  --workspace $fixture `
  --dsw "$fixture\Product.dsw" `
  --source src\control.c `
  --function Control_Update `
  --configuration "Win32 Debug" `
  --project Control `
  --out $out `
  --finalize-dossier

py -m unit_test_runner --json prepare-review --dossier "$out\reports\function_dossier.json"
py -m unit_test_runner --json run-tests --workspace $out --dry-run
py -m unit_test_runner --json prepare-evidence --workspace $out
```

`function_dossier.json` keeps the original analysis fields after finalization and adds dossier review
fields. The same dossier can still be passed to `build-probe --dossier` and
`generate-test-design --dossier`.

## Practical Analyzer Fixture

`tests/fixtures/vc6_practical_project` is a denser sample for analyzer regression and manual
review. It keeps the same CLI flow but targets `DeviceControl_Update`, which includes file-scope
static state, extern globals, arrays, struct members, callback/function-pointer calls, and
`#define` constants, aliases, and function-like macros.

```powershell
$env:PYTHONPATH = "$PWD\src"
$fixture = "$PWD\tests\fixtures\vc6_practical_project"
$out = "$env:TEMP\unitTestRunner-practical\DeviceControl_Update"

py -m unit_test_runner analyze-function `
  --workspace $fixture `
  --dsw "$fixture\Product.dsw" `
  --source src\device_control.c `
  --function DeviceControl_Update `
  --configuration "DeviceControl - Win32 Debug" `
  --project DeviceControl `
  --out $out

py -m unit_test_runner build-probe --dossier "$out\reports\function_dossier.json" --dry-run
```
